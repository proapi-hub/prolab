# 发布公告

以后所有推送公告都更新在根目录 `CHANGELOG.md`。这是 Web 端版本弹窗和文档站[更新日志](/docs/progress/changelog)共同读取的唯一来源。

- 准备中的内容写到 `Unreleased`。
- 正式发布时，把 `Unreleased` 移到 `vX.Y.Z - YYYY-MM-DD` 区块。
- 同步把根目录 `VERSION` 改成同一个版本号。

每个正式版本建议按这几个小节写：

- `版本定位`：说明这个版本解决什么问题，适合谁使用。
- `面向用户的变化`：用普通用户能理解的话说明新增、优化和修复。
- `数据保存说明`：涉及本地存储、同步、导入导出时必须写清楚边界。
- `当前范围`：说明本版本暂不支持的能力，避免用户误解。

不要在其他地方重复维护发布记录。
